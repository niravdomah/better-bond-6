**Story 1: Payment Grids with Park and Unpark Workflows**

Build the Payment Management page with both grids and all park/unpark interactions. On page load, fetch payments for the selected agency via `GET /v1/payments?AgencyName={agency}`, split them into READY (Main Grid) and PARKED (Parked Grid). The Main Grid displays all READY payments with the full column set (Agency Name, Batch ID, Claim Date, Agent Name & Surname, Bond Amount, Commission Type, Commission %, Grant Date, Reg Date, Bank, Commission Amount, VAT, Status). Each row has a Park button; selecting multiple rows enables a "Park Selected" bulk action. Both single and bulk park trigger confirmation modals and call `PUT /v1/payments/park`. The Parked Grid is always visible (even when empty) and mirrors the Main Grid columns, with Unpark and "Unpark Selected" actions that call `PUT /v1/payments/unpark`. After any action, the data refreshes. Includes client-side search/filter bar (by Claim Date, Agency Name, Status), pagination on the Main Grid, loading skeletons, error banner with retry, and en-ZA currency formatting. The `LastChangedUser` header is sent on all mutating calls.

- **Route:** `/payment-management?agency={agencyName}`
- **FRS coverage:** R12, R13, R14, R15, R16, R17, R18, R19, R22, R23, R33, R34, R35
- **API endpoints:** `GET /v1/payments`, `PUT /v1/payments/park`, `PUT /v1/payments/unpark`
- **Estimated ACs:** ~18-20

---

**Story 2: Initiate Payment Flow**

Add the "Initiate Payment" button to the Payment Management page. When clicked, it opens a confirmation modal showing the count of READY payments and their total value for the current agency. On confirmation, call `POST /v1/payment-batches` with all READY payment IDs and the `LastChangedUser` header. On success, show a success modal confirming the batch was processed, and refresh the Main Grid (which should now be empty for this agency). The button is disabled when no READY payments exist. On failure, show an error message with no state change.

- **Route:** `/payment-management?agency={agencyName}`
- **FRS coverage:** R20, R21, R22, R28, R33
- **API endpoints:** `POST /v1/payment-batches`
- **Estimated ACs:** ~8-10

---
